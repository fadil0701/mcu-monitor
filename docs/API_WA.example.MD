## Dapatkan Phone Number ID

```
GET https://chat.api.co.id/api/v1/public/phone-numbers
Authorization: Bearer <API_KEY>
```

Postman: **Account тЖТ List Phone Numbers** тЖТ salin **`data[].id`** (bukan `phone_number_id`).

Contoh response Api.co.id:

```json
{
  "id": "cmqj83dp7ebifo8dyegp1b6ki",
  "phone_number_id": "1081054025088559",
  "display_phone_number": "+62 819-7960-786"
}
```

| Field response | Isi di admin Phone Number ID |
|----------------|------------------------------|
| **`id`** | тЬЕ `cmqj83dp7ebifo8dyegp1b6ki` |
| `phone_number_id` | тЭМ Jangan (ID Meta internal) |
| `waba_id` | тЭМ Jangan |

## Troubleshooting

| Error | Penyebab | Tindakan |
|-------|----------|----------|
| `WhatsApp account not found or not owned by user` | Salah field: pakai **`id`** bukan **`phone_number_id`** dari `/phone-numbers` | Ganti Phone Number ID ke `cmqj83тАж` (bukan `108105тАж`) |
| API Key salah / token lama | Field token dikosongkan saat simpan (token lama tetap) | Pengaturan тЖТ WhatsApp тЖТ isi ulang API Token тЖТ Simpan |
| Nomor hanya di Meta, belum di Api.co.id | Akun WA belum connect ke Chat Gateway | Dashboard Api.co.id тЖТ hubungkan nomor WA (Cloud API) |
| `Template not found` | Nama template salah / belum APPROVED | Sync template di Api.co.id, cocokkan `mcu_undangan` / `mcu_hasil` |

### Verifikasi cepat (Postman)

1. Set `api_key` = API Key dari admin MCU (bukan key akun lain).
2. **Account тЖТ List Phone Numbers** тАФ response harus `success: true` dan ada `data[]`.
3. Salin `data[0].id` (atau yang `is_primary: true`) тЖТ **Phone Number ID** di admin.
4. **Customers тЖТ Create Customer** тАФ body pakai `whatsapp_phone_number_id` yang sama. Harus sukses sebelum kirim dari MCU.
5. **Send Template** тАФ uji kirim ke nomor tes.

Jika langkah 2 kosong atau error тЖТ nomor belum terhubung ke akun Api.co.id yang memiliki API Key tersebut.
